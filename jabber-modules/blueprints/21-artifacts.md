# Artifacts — Centralized Management System Blueprint

**Category:** `ARTIFACTS` | **Slug:** `artifacts` | **Group:** System Infrastructure
**Exemption:** This category is EXEMPT from the Two-Mode Execution requirement. It functions as a centralized listener and manager.

---

## 1. Objective
The **Artifacts System** is the authoritative hub for storing, managing, viewing, and organizing every piece of data generated by the JABBER module suite. It ensures complete visibility, forensic integrity (SHA256), and persistent retrievability of all assessment outputs.

---

## 2. Centralized Storage Architecture
Every module in the suite automatically transmits its results to this system. Data is organized into a hierarchical, structured tree within the module's execution workspace:

### A. Directory Classification
1.  **`reports/artifacts/`**: General forensic data, network captures (`.pcap`), wireless handshakes (`.cap`), screenshots, and evidence files.
2.  **`reports/payloads/`**: Weaponized binaries, scripts, or installers generated by the system (e.g., `.exe`, `.elf`, `.msi`, `.apk`).
3.  **`reports/outputs/`**: Structured data exports and raw tool results (e.g., `.csv`, `.json`, `.xml`, `.txt`, `.log`).
4.  **`reports/analysis/`**: Technical fingerprints and environment dumps generated by the profiling engine.

### B. Universal Naming Convention
To ensure clarity and prevent collisions, all files MUST follow this pattern:
`[Mode]_[Timestamp]_[OriginalName].[Ext]`
*Example:* `EXPLOIT_1714512000_reverse_shell.elf`

---

## 3. Enhanced Profiling System
The profiling engine transforms raw assessment data into high-fidelity "Target Profiles." As of V5, profiles are generated and saved in five concurrent formats:

| Format | Role | Detail Level |
|--------|------|--------------|
| **HTML** | **Primary / Default** | **MAXIMUM**. Interactive, styled, and user-friendly representation of the target. |
| **JSON** | Machine-Readable | High. Used for automation and programmatic ingestion. |
| **Markdown** | Documentation | High. Ready for inclusion in professional security reports. |
| **TXT** | Fast Review | Medium. Plaintext summary for quick terminal viewing. |
| **RAW** | Forensic Audit | Raw. Unprocessed output from the profiling logic. |

---

## 4. Global Tool Management (UI Component)
The Artifacts system hosts the centralized **Global Tool Registry**, which aggregates tools from every module's `ToolManager`.

### UI Specifications:
- **Aggregation**: Scans all active modules to build a master list of binaries.
- **Sorting**: Alphabetical by tool name.
- **Grouping**: Grouped by category (e.g., "Reconnaissance Tools", "Wireless Tools").
- **Association**: Clearly shows which module(s) utilize each tool.
- **Action Controls**:
    - **Download**: Retrieve specific tool binaries for external use.
    - **Delete**: Remove individual tools or versions.
    - **Bulk Manage**: Options to "Update All" or "Clean Unused Tools."
    - **Status Indicators**: Real-time check of binary paths and versions.

---

## 5. Automated Output Lifecycle
Modules do not "choose" to save; the infrastructure enforces it:
1.  **Orchestration**: The `ModuleEngine` executes tools via `ProcessExecutor`.
2.  **Capture**: Every `stdout`/`stderr` is captured and saved as a `.log` artifact.
3.  **Extraction**: Files generated by tools are moved to the appropriate directory (`reports/payloads/`, `reports/artifacts/`, etc.).
4.  **Integrity**: Every file is hashed using **SHA256**.
5.  **Telemetry**: A `CommandRecord` is generated for every CLI execution, linking the command string to its specific output file.
6.  **Persistence**: The Artifacts system indices these files in the `ModuleResult` for immediate UI visibility and long-term storage.

---

## 6. Frontend Integration
The Artifacts system provides the "Artifact Gallery" UI component used by all modules:
- **Previewer**: Real-time viewing of text-based logs and images.
- **Download Manager**: Secure download of payloads and captures.
- **Integrity Badge**: Displays the SHA256 hash for forensic verification.
